1核心论文（必读）

2407.13664 已部署 · 美团
Decision Focused Causal Learning for Direct Counterfactual Marketing Optimization
Hao Zhou, Rongxiao Huang, Shaoming Li, Guibin Jiang, Jiaqi Zheng, Bing Cheng, Wei Lin
美团（Meituan）· 2024-07
DFL 因果推断 营销优化 端到端
一句话总结：将因果推断（ITE估计）和运筹优化（整数随机规划）端到端结合，优惠券分配直接反事实优化，已在美团多场景上线A/B测试有效。

问题建模：用户特征x → 是否发优惠券 → 最大GMV（受预算B约束）
挑战：①0-1整数随机规划（券面额∈{10,20,50,100}）；②反事实损失不可得；③OR求解器调用成本极高
三大技术挑战 & 解决方案
  • 整数随机规划：将预算不确定性建模为随机变量
  • 反事实损失不可得：通过因果推断估计ITE，用ITE代理反事实损失
  • 求解器成本高：采样梯度估计 + 历史数据缓存策略
核心方法
  • 因果估计模块：估计每个用户对每种券的ITE（uplift = E[GMV|do(send)] − E[GMV|do(not)]）
  • DFL决策模块：基于ITE预测做组合优化（top-K用户 × 券面额）
  • 端到端训练：DFL regret指导ITE预测损失更新
✓ 美团外卖/到店多场景 A/B 测试，GMV 提升显著
2507.05510 已部署 · Lightspeed
Heterogeneous Causal Learning for Optimizing Aggregated Functions in User Growth
Shuyang Du, Jennifer Zhang, Will Y. Zou
Lightspeed × 多伦多大学 × Angle.ac · 2025-07
DFL 因果推断 用户增长 ROI优化
一句话总结：跳过uplift显式建模，用Softmax门控直接端到端优化ROI（增量收益/增量成本），配合Barrier Function + Annealing处理硬约束，已全球部署。

核心洞察：传统两阶段方法（先预测uplift再排序）的问题——训练目标 ≠ 业务目标。DRM直接以整体ROI为loss，通过softmax(p_i)构造可微群体级优化。
DRM 核心公式
  • 优化目标：max_θ τ_r*(θ,X) / τ_c*(θ,X)（整体ROI，而非个体uplift）
  • p_i = softmax(f(x_i|θ)) → 可微的概率权重，归一化用户相对重要性
  • 分子 = ΣY_rⁱ · p_i · (I_{T=1} − I_{T=0}) → 加权增量收益
  • 分母 = ΣY_cⁱ · p_i · (I_{T=1} − I_{T=0}) → 加权增量成本
CRM：Barrier Function + Annealing 约束处理
  • σ_i = sigmoid(T · (f(x_i) − d*)) → Sigmoid型Barrier动态"屏蔽"低效用户
  • Annealing：T从小→大（软约束探索→硬约束聚焦），避免梯度断裂
  • 支持比例约束（Top-40%用户）和预算约束（总成本≤$B）
  • Propensity weighting扩展：处理非随机实验数据的selection bias
✓ 线上ROI相比Causal Forest提升超20%，已全球部署于HCLW生产系统

4关键洞察

核心发现
  • DFCL vs HCL 的本质区别：DFCL是两阶段（先因果建模ITE → 再OR求解），HCL是真正单阶段端到端（Softmax群体级优化直接可微）
  • HCL的核心洞察：不追求每个用户的uplift预测准确，而是学准「哪个群体组合整体ROI最大」的打分函数f(x)
  • 反事实不可得的通用解法：用ITE（个体治疗效果）代理反事实损失，这是所有因果DFL方法的共同基础
  • 约束处理的两种范式：① DFCL的随机规划（处理预算不确定性）；② HCL的Barrier+Annealing（处理比例/预算硬约束）
  • 端到端 vs 两阶段的核心矛盾：两阶段方法训练目标（预测MSE）≠ 业务目标（决策ROI），这是端到端DFL兴起的根本原因
  • 工业落地共性：三篇论文均已线上部署，说明因果DFL在营销场景已从学术研究进入工业实践阶段
